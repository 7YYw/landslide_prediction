  项目整体实现流程

  项目分 V1（原有） 和 V2（新增） 两套并行流程，两者共享同一份原始数据，但方法论完全不同。

  ---
  二、V1 流程（原有，pre_process/preprocess.py）

  核心思想

  在已有灾害点内部做二分类：降雨诱发滑坡 → 标签0，其他 → 标签1。

  标签定义

  label = 0  如果 triggering_factor 含"降雨" 且 hazard_type 为"滑坡"
  label = 1  其他情况
  # 分布: 标签0=2144(29%), 标签1=5171(71%)

  数据流

  原始CSV (7315×31)
      │
      ▼
  preprocess_data()  ─── 特征工程 ─── 构建43维特征
      │                                  │
      │  • 基础特征: elevation, aspect,   │
      │    ndvi, ndwi, landuse, rain_*   │
      │  • 交互特征: rain_ratio,         │
      │    valley_risk, aspect_rain等     │
      │  • 地理聚类: KMeans 10类          │
      │  • 缺失标记: ndvi_missing等       │
      ▼                                  ▼
  训练/测试划分 (80/20, 分层采样)
      │
      ▼
  IterativeImputer 缺失值插补
      │
      ▼
  StandardScaler 标准化
      │
      ▼
  返回 (X_train, X_test, y_train, y_test)

  调用链

  # 各模型脚本 → preprocess_data() → 训练 → 保存 models/*.pkl
  ML_algorithms/train_random_forest.py
      └─ from pre_process import preprocess_data
      └─ preprocess_data() → 获取数据
      └─ RandomForestClassifier.fit() → 训练
      └─ joblib.dump(models/random_forest.pkl)

  # 统一入口
  train_models.py
      └─ 遍历 ML_algorithms/*.py 中的9个模型函数
      └─ 遍历 improve_algorithms/*.py 中的4个集成模型函数
      └─ 阈值优化 → 结果排名 → ROC/PR曲线

  # 集成模型
  improve_algorithms/train_stacking.py
      └─ 从 models/*.pkl 加载预训练基模型
      └─ 构建元特征 → 训练元学习器 → 校准

  ---
  三、V2 流程（新增，本次实现）

  核心思想

  外部生成非灾害点作为负样本，实现真正的滑坡易发性二分类（灾害/非灾害），且将暴露特征分离到风险分析阶段。

  模块调用链

  train_models_v2.py (用户入口)
      │
      ├─ pre_process/pipeline_v2.py  (核心编排)
      │   │
      │   ├─ pre_process/data_cleaning.py     阶段1: 数据清洗
      │   ├─ pre_process/imputation.py        阶段2: 缺失值插补
      │   ├─ pipeline_v2.build_features()     阶段3: 特征工程（27维易发性特征）
      │   ├─ pre_process/negative_sampling.py 阶段4: 负样本生成
      │   └─ train_test_split + StandardScaler 阶段5: 划分+标准化
      │
      ├─ 9个基模型训练  (同V1算法，但用新数据)
      ├─ 4个集成模型训练 (同V1算法，但用新数据)
      ├─ [可选] Optuna超参优化 (新增)
      ├─ SHAP特征分析 (新增)
      └─ 结果排名 + 可视化

  hazard_mapping.py (风险分析入口)
      │
      ├─ 从 models_v2/ 加载最佳模型
      ├─ pipeline_v2 获取全量数据
      ├─ 五级危险性分类 (quantile/equal)
      └─ 综合风险 = 易发性 × 暴露度 → 制图

  四、各模块详细流程

  4.1 data_cleaning.py — 数据清洗

  输入: 原始CSV (7315×31)
                     操作                             输出
  ┌──────────────────────────────────────────────────────────────────┐
  │  hazard_type筛选: 滑坡+不稳定斜坡+泥石流                         │
  │  保留5622条，排除崩塌(1689)+地面塌陷(4)              →  5622条   │
  ├──────────────────────────────────────────────────────────────────┤
  │  删除10个冗余字段: id, discovery_date, location, data_source,    │
  │  development_trend(84%缺失), industry(81%缺失),                   │
  │  disaster_type, disaster_scale, scale_grade, industry_transferred│
  │                                                       →  21列   │
  ├──────────────────────────────────────────────────────────────────┤
  │  空值行删除: triggering_factor(66行) + threat_target(75行)       │
  │                                                       →  ~5550条 │
  ├──────────────────────────────────────────────────────────────────┤
  │  异常值IQR过滤: elevation∈[500,5500], aspect∈[0,360]             │
  │  tpi/distance_to_river的1%-99%区间                     →  ~5300条 │
  ├──────────────────────────────────────────────────────────────────┤
  │  重复值删除(lon+lat去重)                                        │
  │  添加缺失指示器(ndvi_missing, rain_missing)          →  最终~5300 │
  └──────────────────────────────────────────────────────────────────┘

  4.2 imputation.py — 缺失值插补

        NDVI (43%缺失)                    降雨 (19.6%缺失)
            │                                  │
            ▼                                  ▼
    RandomForestRegressor               IterativeImputer (MICE)
    n_estimators=200                    estimator=RF, max_iter=30
    max_depth=10                        联合插补: rain_3d/7d/30d
    特征: elevation+tpi+                 +api+elevation+lon+lat
          distance_to_river+
          lon+lat+landuse
            │                                  │
            ▼                                  ▼
    NDWI (43%缺失)                         完成
      2步插补:
      Step1: NDWI ~ NDVI 线性回归
      Step2: 残差用RF预测

  4.3 pipeline_v2.py — 特征工程（核心设计）

  易发性特征（27个，用于分类）

  地形(4)     elevation, aspect, tpi, distance_to_river
  降雨(4)     rain_3d, rain_7d, rain_30d, api
  植被(2)     ndvi, ndwi
  坐标(2)     lon, lat
  土地利用(1)  landuse
              └─ One-hot编码: lu_Trees, lu_Grassland, lu_Built-up等(6个)
  衍生(7):
    rain_ratio_3d_30d    短期降雨集中度
    rain_ratio_7d_30d    中期降雨集中度
    rain_intensity       降雨强度变化率
    valley_risk          沟谷汇水风险 = max(0,-tpi)/(dist_river+1)
    elev_rain            高程×降雨交互 = elevation×log1p(rain_3d)
    aspect_rain          坡向×降雨交互 = cos(aspect)×rain_3d
    ndvi_elev            植被×高程耦合
    log_dist_river       距河距对数变换

  暴露特征（4个，不参与分类，用于风险分析）

  threatened_people      威胁人口
  threatened_households  威胁户数
  threatened_property    威胁财产
  exposure_index         综合暴露指数 = 0.4P + 0.3HH + 0.3Property

  4.4 negative_sampling.py — 负样本生成

  输入: 正样本DataFrame (~5300条灾害点)
             │
             ▼
      ┌─────────────────────┐
      │  BallTree空间索引    │  haversine球面距离
      │  经纬度→弧度        │
      └─────────┬───────────┘
                │
      ┌─────────▼───────────┐
      │  随机采样候选点       │  正样本范围扩展5%
      │  检查距离>500m       │  最多尝试30倍次数
      └─────────┬───────────┘
                │ 通过
      ┌─────────▼───────────┐
      │  最近邻特征继承       │  找最近3个灾害点
      │  加权平均 + 高斯噪声  │  按反距离加权
      └─────────┬───────────┘
                │
      ┌─────────▼───────────┐
      │  暴露度置0           │  非灾害区无人
      │  标签=0             │
      └─────────────────────┘
                │
                ▼
  输出: 负样本DataFrame (N条，ratio×正样本数)

  4.5 train_models_v2.py — 模型训练

  加载数据 (V2 Pipeline)
      │
      ▼
  ┌── 9个基模型 ─────────────────────────────────────────┐
  │  logistic_regression  → 逻辑回归                      │
  │  decision_tree        → 决策树                        │
  │  svm                  → 支持向量机(rbf核)             │
  │  knn                  → K近邻(k=7, distance权重)     │
  │  random_forest        → 随机森林(200棵树)             │
  │  gboost               → 梯度提升(200轮, lr=0.1)      │
  │  xgboost              → XGBoost(正负比权重)          │
  │  lightgbm             → LightGBM(balanced权重)       │
  │  catboost             → CatBoost(Balanced权重)       │
  └──────────────────────────────────────────────────────┘
      │
      ▼
  ┌── 4个集成模型 ────────────────────────────────────────┐
  │  ensemble_avg   → 9模型概率算术平均                    │
  │  voting         → VotingClassifier(软投票)            │
  │  stacking       → StackingClassifier(5折CV+LR元模型) │
  │  blending       → hold-out验证集+LR元模型             │
  └──────────────────────────────────────────────────────┘
      │
      ▼  (可选 --optuna-trials N)
  ┌── Optuna超参优化 ─────────────────────────────────────┐
  │  random_forest_tuned  贝叶斯调参30-50次               │
  │  xgboost_tuned        搜索学习率/树深/正则化          │
  │  lightgbm_tuned       搜索叶子数/子采样/正则化         │
  │  catboost_tuned       搜索深度/学习率/L2正则           │
  └──────────────────────────────────────────────────────┘
      │
      ▼
  ┌── 后处理 ─────────────────────────────────────────────┐
  │  CalibratedClassifierCV(sigmoid)  概率校准             │
  │  阈值搜索 (0.05-0.95, 步长0.01)   多目标优化           │
  │  目标: 0.6×F1 + 0.4×特异性                            │
  └──────────────────────────────────────────────────────┘
      │
      ▼
  ┌── 输出 ───────────────────────────────────────────────┐
  │  results_v2/model_performance.csv  13+模型排名         │
  │  results_v2/roc_curves.png         ROC曲线             │
  │  results_v2/pr_curves.png          PR曲线              │
  │  results_v2/shap_summary.png       SHAP特征重要性      │
  │  models_v2/*.pkl                   训练好的模型         │
  └──────────────────────────────────────────────────────┘

  4.6 hazard_mapping.py — 危险性分级

  输入: 最佳模型 + pipeline_v2全量数据
      │
      ▼
  ┌── 易发性预测 ──────────────────────────────────────────┐
  │  model.predict_proba(X_full)[:,1]                      │
  │  输出: proba ∈ [0, 1]                                 │
  └──────────────────────────────────────────────────────┘
      │
      ▼
  ┌── 五级危险性分类 ──────────────────────────────────────┐
  │  方法1: quantile (五分位数, 每级20%)                    │

      │
      ▼
  ┌── 综合风险 = 易发性 × 暴露度 ──────────────────────────┐
  │  Risk = P_sus × (0.4P + 0.3HH + 0.3Property)          │
  │  等级: 极低(<0.001) / 低(0.001-0.01) / 中 / 高 / 极高 │
  └──────────────────────────────────────────────────────┘
      │
      ▼
  ┌── 可视化输出 ─────────────────────────────────────────┐
  │  susceptibility_distribution.png   概率分布直方图        │
  │  hazard_classification.png        五级空间散点图        │
  │  risk_map.png                    风险空间分布图         │
  │  hazard_analysis_dashboard.png   综合分析仪表盘          │
  │  hazard_breakdown.csv            各等级统计汇总         │
  │  hazard_prediction_full.csv      全量预测数据           │
  └──────────────────────────────────────────────────────┘
